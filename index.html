ทำไมเราต้องเปลี่ยน? (Pain Points)

ปัญหาที่เราเผชิญอยู่ทำให้ประสิทธิภาพการผลิตลดลงและมีความเสี่ยงต่อความผิดพลาดของข้อมูล

Manual Process

การบันทึกด้วยมือทำให้ข้อมูลสูญหาย หรือผิดพลาดได้ง่าย

Lack of Visibility

ไม่ทราบสถานะเครื่องจักร Real-time ต้องเดินลงไปดูหน้างาน

Communication Gap

การส่งต่อข้อมูลระหว่างฝ่ายวางแผนและหน้าเครื่องล่าช้า

Approval Delay

การอนุมัติก่อนเดินเครื่องใช้เวลานานและตรวจสอบยาก

ภาพรวมระบบ (System Overview)

Streamlit Framework

Frontend ทันสมัย รองรับการแสดงผลข้อมูลซับซ้อนอย่างรวดเร็ว

Supabase Database

ฐานข้อมูล Real-time Cloud มั่นใจได้ในความปลอดภัยและความไว

Anywhere Access

ระบบทำงานอย่างไร? (Workflow)

1

Planning (วางแผน)

Planner สร้างใบสั่งงาน (SO) ระบุสินค้าและ Spec ผ่านระบบส่วนกลาง

2

Assignment (รับงาน)

ระบบแจ้งเตือน Operator (M1-M9) ทันทีบนอุปกรณ์ประจำเครื่อง Operator กด "เลือกทำ"

3

Double Validation (ตรวจสอบ 2 จุด)

Feeder: ยืนยัน Spec เหล็กต้นทาง
Stacker: ยืนยันการรับ/พลิกเหล็ก
4

Approval (อนุมัติ)

Supervisor ตรวจสอบความถูกต้องผ่านมือถือส่วนตัว และกด "Approve"

Execution (ผลิต)

ปุ่ม START ปรากฏขึ้นบนเครื่องจักร Operator จึงจะเริ่มเดินเครื่องได้

ฟีเจอร์เด่น (Key Features)

Safety Locking

ล็อคปุ่ม Start จนกว่าจะผ่านการยืนยัน

Real-time Dashboard

เห็นสถานะเครื่องจักรสดๆ Running/Stop

Smart Alerts

แจ้งเตือนทันทีเมื่อมีงานหรือขออนุมัติ
